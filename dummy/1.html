팀에 SSO(Single Sign-On)를 사용 설정하는 방법 Advanced 또는 Enterprise 요금제를 사용하는 팀의 관리자는 팀에 SSO(Single Sign-On)를 사용 설정할 수 있습니다. SSO(Single Sign-On)를 사용하면 팀원들이 통합 ID 공급업체에 로그인하여 에 액세스할 수 있습니다. 즉, 팀원들이 다른 비밀번호를 기억하고 있지 않더라도 에 액세스할 수 있습니다. SSO(Single Sign-On)를 사용 설정하는 방법 해당 ID 공급업체의 사이트에서 SSO(Single Sign-On) 구성 안내에 따릅니다. 많은 대형 ID 공급업체에서 에 사전 구성된 설정을 제공합니다. 해당 ID 공급업체가 지원되지 않는 경우에는 SSO(Single Sign-On)를 위한 자체 ID 공급업체 솔루션을 구성할 수 있습니다. X.509 인증서 사본을 다운로드하고 로그인 URL을 메모하여 8단계에서 사용합니다. 관리자 자격 증명으로 x에 로그인합니다. 사이드바에서 관리 콘솔을 클릭합니다. 사이드바에서 설정을 클릭합니다. 인증에서 SSO(Single Sign-On)를 선택합니다. SSO(Single Sign-On) 설정을 해제에서 선택 사항 또는 필수 항목으로 변경합니다. 필수 항목을 선택한 경우 팀원들은 SSO를 사용해 에 로그인해야 하며, 비밀번호는 더 이상 사용할 수 없습니다. 단, 관리자는 기존대로 관리자 로그인 정보로 로그인할 수 있습니다. 선택 사항을 선택한 경우 팀원들은 SSO 또는 비밀번호를 사용해 에 로그인할 수 있습니다. 로그인 URL 추가를 클릭한 다음 2단계에서 메모한 URL을 입력합니다. 선택 사항: 로그아웃 URL 추가를 클릭하여 로그아웃 URL을 추가할 수 있습니다. 인증서 업로드를 클릭하여 앞서 다운로드한 X.509 인증서 .pem 파일을 업로드합니다. 변경사항 적용을 클릭합니다. 팀원들에게 알립니다. SSO를 필수 항목으로 선택한 경우 에서 팀원들에게 이메일로 알립니다. SSO를 선택 사항으로 선택한 경우 팀원들에게 직접 알려야 합니다. SSO(Single Sign-On)를 사용 설정한 후 나머지 팀원들을 위해 이러한 안내를 공유할 수 있습니다. 계정에 연결되어 있는 모든 장치는 계속해서 작동합니다. 이제 해당 ID 공급업체에서 비밀번호 및 로그인 관련 보안을 관리하므로 관리자는 더 이상 를 통해 비밀번호를 재설정하거나 2단계 인증을 요구할 수 없습니다. 팀의 사용자 지정 SSO 로그인 URL을 찾는 방법 팀원들이 해당 ID 공급업체에 이미 로그인한 상태라면 사용자 지정 링크를 사용해서 각자의 계정에 바로 이동할 수 있습니다. 팀의 사용자 지정 SSO 로그인 URL을 찾으려면 다음과 같이 하세요. 관리자 자격 증명으로 에 로그인합니다. 관리 콘솔을 클릭합니다. 설정을 클릭합니다. SSO(Single Sign-On)를 클릭합니다. SSO 로그인 URL 섹션에서 링크 복사를 클릭합니다.
